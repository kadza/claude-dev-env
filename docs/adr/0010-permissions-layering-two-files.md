# Permissions layering: two files, no merge tooling

`~/.claude/settings.json` is a symlink to `general/settings.json` (git commands, `Skill(*)`, universal tools); each template also ships a starter `.claude/settings.local.json` pre-filled with that tech's allowlist (e.g. npm/vitest for node-ts, uv/pytest for python), gitignored by convention and the natural target for in-session approvals. Settings are plain JSON with no `@import` mechanism, and Claude Code only has user + project slots — no framework slot — so this keeps live write-back on both files. Rejected: folding every framework's permissions into general (every container would allow every tech's commands); jq-merging into a generated user settings file (approvals would land in a generated file that never reaches the repo).
