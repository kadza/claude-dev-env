# Screenshots reach Claude as files, not clipboard paste

Claude Code's clipboard image paste doesn't work inside the devcontainers: the CLI reads images via `wl-paste`/`xclip`/`xsel`, which need an X11/Wayland selection, and these containers have neither the binaries nor a display server. Instead, macOS screenshots go to a host folder that's bind-mounted into the container, and Claude reads the newest file — files are the one channel that crosses the host/container boundary reliably. Rejected: true `Cmd+V` paste via a headless Xvfb + `xclip` baked into every image, plus a standing host-side LaunchAgent pushing clipboard changes via `docker exec` (an always-on background process, a much bigger lift, deferred unless the file flow proves too annoying); a hotkey that dumps the clipboard to a file on demand (works, no daemon, but still a manual per-image trigger that doesn't ride the native screenshot flow).
