# Screenshot destination is a non-default, selectable "Save to" entry

`~/claude-shots` is added as a non-default, selectable destination in the `Cmd+Shift+5` screenshot toolbar's "Save to" menu via a one-time `Options → Save to → Other Location… → ~/claude-shots` pick; the Desktop stays the default, and the user picks `claude-shots` per screenshot when they want Claude to see it. The goal was a destination inside the native screenshot menu, not a global redirect of normal screenshot habits. This isn't scriptable — the only documented knob (`com.apple.screencapture location`) sets the *default* and would redirect every screenshot, which was explicitly rejected — so the pick is a documented one-time manual step. Rejected: setting it as the scriptable default (hijacks the default, Desktop screenshots stop); mounting `~/Desktop` read-only (exposes the whole Desktop, reads from clutter); a Folder Action/watcher copying Desktop shots over (more moving parts to maintain).
